# LogiCore — правила для ИИ-агентов в этом репозитории

LogiCore — логистическая платформа **В ПРОДАКШНЕ** (заявки, трекинг, финансы,
документы). Монорепо pnpm: `apps/api` (NestJS + Prisma), `apps/web`
(Next.js 14 + Ant Design + собственный скин), `apps/mobile` (Expo).

## Железные правила (продакшн)

1. Перед КАЖДЫМ `git commit`: `npx tsc --noEmit` в `apps/web`, и в `apps/api`
   если менялся api. Красный tsc = коммитить ЗАПРЕЩЕНО. Без исключений,
   даже для «мелких» правок.
1а. Один коммит = одна страница/одна задача. Не накапливать несколько страниц
   в один коммит.
1б. Не выдумывать вызовы функций/хелперов. Если нужного хелпера нет в кодовой
   базе — оставь код как был. При редизайне форм поля, их типы (Input/Select)
   и правила валидации не менять.
2. При работе над дизайном/вёрсткой ЛОГИКА НЕ ТРОГАЕТСЯ — меняются только
   JSX-разметка и CSS. Обработчики, запросы, стейт остаются как есть.
3. Миграции БД (`prisma/`) — только по явной просьбе пользователя.
4. Не коммитить и не читать вслух `.env` и другие секреты.
5. Общение и отчёты — на русском.

## Дизайн — обязательно к соблюдению

Перед ЛЮБОЙ работой над интерфейсом сначала прочитай `REDESIGN_PLAN.md` в корне —
это источник правды по дизайну платформы. Краткая выжимка:

### Платформа (кабинеты /company, /admin) — светлый «Apple-стиль»
- Фон страницы `#f4f5f7`, карточки белые, радиус 20–24px, мягкие тени.
- Тёмные акцент-блоки (карты, тикер, featured): глубокий графит `#0b0d12`.
- Акцентный синий `#1677ff`. Шрифт Inter. Суммы — жирные, `tabular-nums`.
- Навигация и фильтры — «пилюли» (активная — чёрная с белым текстом).
- Ant Design НЕ выпиливать: он остаётся движком форм/модалок/таблиц,
  визуально перекрывается классами `lc-*` в `apps/web/src/app/globals.css`.
- Новые UI-библиотеки НЕ добавлять.

### Лендинг (`apps/web/src/app/page.tsx` + `page.module.css`) — тёмный редакционный
- Фон `#030712`, шрифты Unbounded (заголовки) + Inter (текст).
- Фирменные приёмы: индексы секций «(0N — Название)», кинетические заголовки
  с посимвольной анимацией, marquee-строка, гигантский вордмарк в футере,
  кастомный курсор, magnetic-кнопки. Новые секции — строго в этом языке.

### Общее
- Переиспользуй готовые компоненты из `apps/web/src/components/ui/`
  (StatusPill, LiveTicker, FeaturedOrderCard, InteractiveCard, Reveal,
  AnimatedCounter и др.) вместо написания новых аналогов.
- Все шрифты обязаны поддерживать кириллицу.
- Весь пользовательский текст интерфейса — на русском.

## Код

- Страницы `company/orders`, `company/orders/[id]`, `company/users` уже
  перегружены (~2000 строк). НЕ раздувай их дальше: новый UI выноси
  в компоненты, новую логику — в `lib/` или хуки.
- Не плоди `any` — типизируй ответы API через существующие типы в `lib/api`.
- Стили статусов/ролей уже описаны (statusColors, statusLabels, StatusPill) —
  не дублируй, импортируй.

## Команды

- Дев-сервер: `pnpm dev` (api + web) из корня; только веб: `pnpm dev:web`.
- Проверки: `npx tsc --noEmit` в apps/web и apps/api; `pnpm lint`.
- Пакетный менеджер — только pnpm (не npm/yarn).
